# Концепция CRM для рекрутера

## Обзор

Приложение Change Client Hub Mobile переработано в полноценную CRM-систему для рекрутеров. Основная идея: рекрутер ведет всех своих кандидатов в приложении и управляет доступом к ним.

## Основные изменения в концепции

### 1. Централизованная база кандидатов

- **Раньше**: Кандидаты были привязаны к конкретным вакансиям и клиентам
- **Теперь**: Рекрутер ведет единую базу всех своих кандидатов
- Кандидаты могут быть:
  - В общей базе (еще не привязаны к вакансиям)
  - Привязаны к одной или нескольким вакансиям
  - В шортлисте для быстрого доступа
  - В избранном

### 2. Управление доступом

Рекрутер контролирует, каких кандидатов видят клиенты:
- Может отправить выбранных кандидатов конкретному клиенту
- Клиент видит только тех кандидатов, которых ему отправил рекрутер
- Можно отправлять одного кандидата нескольким клиентам (для разных вакансий)

### 3. Статусы кандидатов

Полный цикл работы с кандидатом:

1. **Ожидает отправки** - Новый кандидат в базе, еще не отправлен клиенту
2. **Отправлен заказчику** - Клиент получил доступ к профилю
3. **Интервью у заказчика** - Запланировано или проходит интервью
4. **Проходит тестовое** - Выполняет тестовое задание
5. **Сбор рекомендаций** - Проверка референсов и СБ
6. **Стажировка** - Испытательный период
7. **Оффер** - Предложение о работе сделано
8. **Найм** - Кандидат принят на работу
9. **Отказ** - Кандидат не подошел
10. **Кадровый резерв** - Сильный кандидат на будущее
11. **Уволен** - Нанятый кандидат покинул компанию

## Функциональность списка кандидатов

### Поиск и фильтрация

- **Поиск по:**
  - Имени кандидата
  - Должности
  - Навыкам и технологиям
  - Сфере деятельности

- **Фильтры:**
  - По вакансии (все / конкретная вакансия)
  - По статусу (все / конкретный статус)
  - По локации

### Карточка кандидата

Каждая карточка содержит:

1. **Заголовок:**
   - Чекбокс для множественного выбора
   - Кнопка "В шортлист"
   - Меню дополнительных действий

2. **Основная информация:**
   - Аватар с цветным бордером (цвет = статус)
   - Индикатор "NEW" для новых кандидатов
   - AI Score (процент соответствия вакансии)
   - Название позиции (кликабельно)
   - Имя, локация, возраст
   - Зарплатные ожидания

3. **История работы:**
   - Последнее место работы
   - Период работы
   - Ссылка "еще N" для просмотра всей истории

4. **Быстрые действия:**
   - Отправить (клиенту)
   - Написать сообщение
   - Позвонить

5. **Нижние действия:**
   - Добавить в избранное
   - Посмотреть историю взаимодействий

### Множественный выбор

Возможность выбрать несколько кандидатов и применить групповые действия:
- Отправить клиенту
- Добавить в шортлист
- Отклонить

### Режимы отображения

1. **Список** - Детальные карточки с полной информацией
2. **Kanban** - Визуальная доска со статусами для drag & drop

## Преимущества новой концепции

### Для рекрутера:

1. **Единая база данных** - Все кандидаты в одном месте
2. **Повторное использование** - Один кандидат для нескольких вакансий
3. **Контроль доступа** - Рекрутер решает, кто что видит
4. **История** - Полная история взаимодействий с каждым кандидатом
5. **AI-анализ** - Автоматическая оценка соответствия

### Для клиента:

1. **Готовая выборка** - Видит только подходящих кандидатов
2. **Качество** - Рекрутер отфильтровал базу
3. **Фокус** - Не нужно просматривать сотни резюме

## Workflow рекрутера

### 1. Добавление нового кандидата

```
Кандидат найден → Добавлен в базу → Статус "Ожидает отправки"
                                   ↓
                              AI анализ
                                   ↓
                         Оценка соответствия
```

### 2. Работа с вакансией

```
Новая вакансия → Поиск в базе → Фильтрация по навыкам
                                        ↓
                                  Формирование шортлиста
                                        ↓
                                Отправка клиенту
```

### 3. Отслеживание прогресса

```
Kanban доска → Drag & Drop → Автоматическое обновление статуса
                                        ↓
                                  История изменений
                                        ↓
                                   Уведомления
```

## Технические особенности

### Структура данных

```typescript
interface Candidate {
  id: string;
  name: string;
  position: string;
  location: string;
  skills: string[];
  status: CandidateStatus;

  // CRM-специфичные поля
  shortlisted: boolean;      // В шортлисте
  isNew?: boolean;           // Новый кандидат
  companyId?: string;        // Для каких компаний доступен
  vacancyId?: string;        // К каким вакансиям привязан

  // AI и аналитика
  aiAnalysis?: AIAnalysis;   // Оценка соответствия

  // История
  history: Interaction[];     // Все взаимодействия
}
```

### Фильтрация и поиск

- Поиск работает в реальном времени
- Фильтры комбинируются (AND логика)
- Результаты кешируются через `useMemo`

### Производительность

- Виртуализация списка через FlatList
- Оптимистичные обновления UI
- Ленивая загрузка деталей
